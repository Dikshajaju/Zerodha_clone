##  Project Highlights

- Monorepo architecture with frontend, backend, and dashboard
- Clean separation of concerns between UI, APIs, and database
- RESTful API design
- Secure handling of environment variables
- Scalable and maintainable project structure

---

##  Application Flow

1. User interacts with the Frontend UI
2. Frontend sends API requests to Backend
3. Backend processes business logic and communicates with MongoDB
4. Backend sends responses to Frontend and Dashboard
5. Dashboard displays portfolio, holdings, and analytics data

##  Module Breakdown

### Frontend Module
- User authentication UI

- Portfolio overview

### Backend Module

- MongoDB integration using Mongoose
- Business logic for orders, holdings, and positions
- Centralized error handling
- Environment variable configuration
